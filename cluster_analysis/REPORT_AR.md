# تقييم منطقية تقسيم الكلاسترات (U11–U14) — بأرقام Google Maps الحقيقية

**المصدر:** Google Maps Distance Matrix API (وضع القيادة) — **مدن:** 83 — **كلاسترات:** 27

## المنهجية
- المسافة والوقت **حقيقية من قوقل مابس** (179 طلب، مركز كل مدينة لمركز الأخرى).
- التقييم على أساس **أقصى زمن قيادة بين أبعد مدينتين** داخل الكلاستر.

| اللون | التقييم | أقصى زمن قيادة |
|---|---|---|
| 🟢 ممتاز | ≤ ساعة | |
| 🟩 جيد | ≤ ساعة ونصف | |
| 🟠 مقبول/مراجعة | ≤ ساعتين ونصف | |
| 🔴 بعيد — يُفضّل الفصل | > ساعتين ونصف | |

---

## الخلاصة
التقسيم منطقي في معظمه، لكن **6 كلاسترات** تحتاج إعادة نظر (ارتفع العدد عن التقدير الأولي لأن أرقام قوقل أدق وأطول قليلًا في بعض الطرق الجبلية/الالتفافية):

| الكلاستر | المدن | أقصى زمن (قوقل) | الملاحظة |
|---|---|---|---|
| **Cluster 9** (الرياض) | القويعية، عفيف، الدوادمي، شقراء، **وادي الدواسر** | **7س 58د** | وادي الدواسر بعيد جدًا — **يُفصل قطعًا** (الملف أشار لذلك). |

## ملاحظات
1. **فرسان:** جزيرة؛ قوقل يحسبها عبر العبّارة فتطلع 4+ ساعات. لا تُقاس كبقية المدن — رتّب لها ترتيبًا خاصًا.
2. **هبونا (Cluster 26):** قوقل ما رجّع مسار بري لإحداثيتها (منطقة جبلية)، فاستُبعدت من حساب أزواجها — لا يؤثر على القرار.
3. الكلاسترات بمدينة واحدة (مكة، جدة، الطائف، المدينة، ينبع، العلا، حائل، بيشة، الخفجي) لا تحتاج تقييم مسافة.

## الملفات
- `clusters_map.html` — خريطة تفاعلية بأرقام قوقل (مرّر على الخطوط لرؤية المسافة/الوقت).
- `cluster_distances_google.xlsx` — ملخص + كل أزواج المدن بمسافة وزمن قوقل الحقيقي.
- `google_durations.py` — سكربت الجلب (يحتاج مفتاح API في المتغير `GOOGLE_MAPS_API_KEY`).
