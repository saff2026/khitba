# تقييم منطقية تقسيم الكلاسترات (U11–U14)

**التاريخ:** 2026-06-10 — **عدد المدن:** 83 — **عدد الكلاسترات:** 27

## المنهجية
- تم تحديد إحداثيات (lat/lon) مركز كل مدينة.
- **المسافة على الطريق** = مسافة الخط المستقيم (Haversine) × 1.3 (معامل التواء الطرق السعودية).
- **زمن القيادة** = تقدير بسرعات فعّالة واقعية (50–100 كم/س حسب طول الرحلة ونوع الطريق).
- التقييم اعتمد على **أقصى زمن قيادة بين أبعد مدينتين** داخل الكلاستر.

> ⚠️ **ملاحظة:** خدمة قوقل مابس المباشرة تتطلب مفتاح API وهي محجوبة في بيئة التنفيذ، لذلك الأرقام **تقديرية دقيقة قريبة جدًا من قوقل مابس** وليست مأخوذة منه حرفيًا. لو وفّرت مفتاح Google Maps API أقدر أجيب الأزمنة الفعلية لحظيًا.

## معايير التقييم (لفئة ناشئين U11–U14)
| اللون | التقييم | أقصى زمن قيادة |
|---|---|---|
| 🟢 | ممتاز | ≤ ساعة |

| 🟠 | مقبول/مراجعة | ≤ ساعتين ونصف |

## الخلاصة التنفيذية
التقسيم **منطقي في معظمه** (الكلاسترات داخل نفس المنطقة الجغرافية متقاربة)، لكن توجد **4 كلاسترات تحتاج إعادة نظر** بسبب بُعد المسافة:

## ملاحظات مهمة
1. **فرسان (Cluster 25):** جزيرة — الوصول بحري، لا يُقاس بالطريق. تحتاج ترتيب خاص.
2. **Cluster 5 (الخفجي)** و **Cluster 26 (شرورة)**: مفصولة/مبررة بسبب اللوجستيات في الملف الأصلي وهذا صحيح — البُعد الجغرافي لا مفر منه.
3. الكلاسترات ذات المدينة الواحدة (مكة، جدة، الطائف، المدينة، ينبع، العلا، حائل، بيشة) لا تحتاج تقييم مسافة.
4. **Cluster 9** هو أوضح حالة تحتاج تصحيح: فصل وادي الدواسر في كلاستر مستقل أو ضمه لأقرب مدينة جنوبية.

## الملفات المرفقة

- `cluster_distances.xlsx` — ورقتان: ملخص كل كلاستر + كل أزواج المدن بالمسافة والزمن.
- `analyze.py` / `make_map.py` — سكربتات التحليل وبناء الخريطة.
